## Rules

- Standalone components — `standalone: true`, `ChangeDetectionStrategy.OnPush`
- Selector prefix — always `ap-` (production and doc site alike)
- Angular 17 control flow — `@if` / `@for`, never `*ngIf` / `*ngFor`
- Tailwind classes in templates for layout and token-backed colour classes only
- CSS tokens (`var(--*)`) in component `.css` files for all spacing, shadows, transitions
- Input names derived from React prop names
- Icons — `icon-outlined` / `icon-solid` / `icon-outlined-new` CSS classes; Material Symbols Rounded

## Component inputs quick reference

### ap-page-layout
Slots: `topNav`, `mobileNav`, `sideNav`, `pageContent`

### ap-top-navigation
Inputs: `appTitle` (string), `logoSrc` (string), `homeHref` (string), `verified` (boolean)
Outputs: `menuOpen`, `notificationsClick`
Slots: `navLinks`

### ap-side-navigation
Inputs: `collapsed` (boolean)
Outputs: `collapseChange`

### ap-side-navigation-mobile
Inputs: `open` (boolean)
Outputs: `closeDrawer`

### ap-sidenav-item
Inputs: `title` (required), `icon`, `iconVariant` (`outlined`|`outlined-new`|`solid`), `href`, `active` (boolean), `expandable`, `expanded`
Outputs: `itemClick`, `expandedChange`

### ap-breadcrumb
Inputs: `crumbs` (array of `{ label: string, href?: string }`)

### ap-dropdown
Inputs: `align` (`left` | `right`, default `left`)
Slots: `trigger` (attribute), panel content

### ap-navigation-profile
Inputs: `avatarUrl` (string), `verified` (boolean)

### ap-logo
Inputs: `variant` (`dark`|`light`|`white`, default `dark`), `width` (string, default `175px`), `iconOnly` (boolean, default `false`)
Assets: `public/logo/icon-color.svg`, `icon-white.svg`, `wordmark-dark.svg`, `wordmark-light.svg`

### ap-button
Inputs: `color` (`primary`|`secondary`|`positive`|`destructive`|`text`), `size` (`md`|`sm`), `disabled`, `type`, `leadingIcon`, `trailingIcon`
Outputs: `clicked`

### ap-tab-navigation
Inputs: `tabs` (`TabItem[]` — `{ label, value?, badge?, icon? }`), `activeIndex` (number)
Outputs: `tabChange` (emits index)

## Doc site helpers quick reference

> Full build guide: **`DOC-PAGES.md`**

### ap-doc-page
No inputs — scroll container wrapper only (title/header removed)
Usage: outermost wrapper for a component doc page

### ap-doc-section
Inputs: `title` (required)
Usage: a card block with a group header — wraps `ap-demo` and `ap-code-block` children

### ap-demo
No inputs — flex row container for `ap-demo-group` children; vertical dividers between groups added automatically

### ap-demo-group
Inputs: `label` (string), `column` (boolean), `gap` (string CSS token)
Usage: padded column cell inside `ap-demo`

### ap-code-block
Inputs: `code` (required string)
Usage: syntax-highlighted HTML at the bottom of a doc section — always last child in `ap-doc-section`

### ap-coming-soon
Inputs: `title`, `description`
Usage: placeholder for unbuilt sections
